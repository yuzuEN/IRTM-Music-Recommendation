# Reranking 是什麼？與圖的關係

## 🤔 Reranking 是什麼？

**Reranking（重排序）** = 在已經找到的候選歌曲中，用**更細緻的方法**重新排序。

---

## 📊 完整流程解釋

### Stage 1: BM25 檢索（找出候選）

```
貼文："I'm feeling sad and lonely"
    ↓
BM25 計算與所有 30,246 首歌的相似度
    ↓
找出 top-100 候選（只考慮「文字相似度」）
```

**結果**：100 首候選歌曲，按 BM25 分數排序
```
歌曲 A: BM25 = 0.95
歌曲 B: BM25 = 0.92
歌曲 C: BM25 = 0.90
...
歌曲 Z: BM25 = 0.75
```

**問題**：只考慮了「文字相似度」，沒有考慮「情緒」和「主題」

---

### Stage 2: Reranking（重新排序）⭐

**目的**：在 top-100 候選內，用 Emotion + Topic 重新排序

**方法**：計算融合分數
```
最終分數 = w1 × BM25分數
        + w2 × 情緒相似度（cosine）
        + w3 × 主題相似度（cosine）
```

**例子**：
```
候選歌曲 A:
  - BM25 = 0.95（文字很相似）
  - 情緒相似度 = 0.60（情緒不太符合）
  - 主題相似度 = 0.50（主題不太符合）
  - 融合分數 = 0.5×0.95 + 0.3×0.60 + 0.2×0.50 = 0.755

候選歌曲 B:
  - BM25 = 0.85（文字較不相似）
  - 情緒相似度 = 0.95（情緒非常符合！）
  - 主題相似度 = 0.90（主題非常符合！）
  - 融合分數 = 0.5×0.85 + 0.3×0.95 + 0.2×0.90 = 0.890

→ 重新排序後：B 排名比 A 高！
```

**為什麼需要 Reranking？**
- BM25 只看「文字」，但貼文的情緒和主題也很重要
- 例如："I'm tired" 可能是工作疲憊（壓力主題），也可能是分手疲憊（感情主題）
- Reranking 可以找到「情境更符合」的歌曲

---

### Stage 3: PPR 在圖上擴散

**目的**：從 Reranking 的高分歌曲開始，在圖上擴散，找到更多相關歌曲

**方法**：
```
以 Reranking 的前 10 首高分歌曲為 "seed"（起點）
    ↓
在圖上做 Personalized PageRank
    ↓
分數會擴散到這些歌曲的鄰居
    ↓
最終推薦列表
```

**例子**：
```
Reranking 後的前 3 名：
  - 歌曲 X（分數 = 0.95）
  - 歌曲 Y（分數 = 0.92）
  - 歌曲 Z（分數 = 0.88）

以 X, Y, Z 為 seed，在圖上做 PPR：
    ↓
歌曲 X 的 20 個鄰居也會得到分數
歌曲 Y 的 20 個鄰居也會得到分數
歌曲 Z 的 20 個鄰居也會得到分數
    ↓
即使某首歌不在 top-100 候選內，
但如果它和 X, Y, Z 相似，也會被推薦！
```

## 🔗 Reranking 和圖的關係

### 它們是**分開的步驟**，但**互相配合**：

```
┌─────────────────────────────────────┐
│ Stage 1: BM25                        │
│ 找出 top-100 候選（文字相似）        │
└─────────────────────────────────────┘
              ↓
┌─────────────────────────────────────┐
│ Stage 2: Reranking                  │
│ 在 top-100 內重新排序                │
│ （考慮 Emotion + Topic）             │
│                                      │
│ 輸出：重新排序後的候選列表            │
└─────────────────────────────────────┘
              ↓
┌─────────────────────────────────────┐
│ Stage 3: PPR（使用圖）               │
│ 以 Reranking 的高分歌曲為 seed       │
│ 在圖上擴散                           │
│                                      │
│ ← 這裡才用到圖！                     │
└─────────────────────────────────────┘
```

### 關鍵點：

1. **Reranking 不需要圖**
   - 只在 top-100 候選內計算
   - 只用到 Emotion 和 Topic 向量
   - 不涉及圖結構

2. **圖需要 Reranking 的結果**
   - PPR 需要知道「哪些歌曲是 seed」
   - Seed 通常來自 Reranking 的高分歌曲
   - 沒有 Reranking，PPR 就不知道從哪裡開始

3. **它們解決不同的問題**
   - **Reranking**：在候選內找到「情境最符合」的歌曲
   - **PPR + 圖**：找到「相關歌曲的相關歌曲」（擴展推薦範圍）

---

## 📝 具體例子

### 貼文："I'm tired after work"

#### Stage 1: BM25
```
找出 top-100 候選（包含 "tired", "exhausted", "work" 等詞的歌）
```

#### Stage 2: Reranking
```
在 top-100 內，計算：
  - BM25 分數（文字相似）
  - 情緒相似度（sadness, neutral）
  - 主題相似度（work, stress）

重新排序後：
  1. 歌曲 A: 融合分數 = 0.92（工作壓力主題，sadness 情緒）
  2. 歌曲 B: 融合分數 = 0.88（工作疲憊主題，neutral 情緒）
  3. 歌曲 C: 融合分數 = 0.85（一般疲憊，但主題不太符合）
  ...
```

#### Stage 3: PPR（使用圖）
```
以歌曲 A, B, C 為 seed
    ↓
在圖上擴散：
  - A 的鄰居（20 首相似歌曲）得到分數
  - B 的鄰居（20 首相似歌曲）得到分數
  - C 的鄰居（20 首相似歌曲）得到分數
    ↓
最終推薦：
  - 歌曲 A: PPR = 0.15（直接相關）
  - 歌曲 A1: PPR = 0.08（A 的鄰居，間接相關）
  - 歌曲 B: PPR = 0.12（直接相關）
  - 歌曲 B1: PPR = 0.06（B 的鄰居，間接相關）
  ...
```

---

## 🎯 總結

### Reranking 是什麼？
- **在候選內重新排序**
- 考慮 Emotion + Topic，不只是文字
- **不需要圖**

### Reranking 和圖的關係？
- **分開的步驟**，但**互相配合**
- Reranking 提供「seed」給 PPR
- PPR 在圖上擴散，找到更多相關歌曲

### 為什麼需要兩個？
- **Reranking**：找到「情境最符合」的歌曲（精準）
- **PPR + 圖**：找到「相關歌曲的相關歌曲」（多樣化）
