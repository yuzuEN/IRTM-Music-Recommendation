# Personalized PageRank (PPR) 分數計算說明

## 🎯 PPR 是什麼？

**Personalized PageRank (PPR)** = 在圖上做「隨機游走」，從特定的「種子節點」（seed）開始，計算每個節點的重要性分數。

---

## 📊 基本概念

### 傳統 PageRank vs Personalized PageRank

**傳統 PageRank**：
- 從**所有節點**開始（均勻分布）
- 計算每個節點的「全局重要性」

**Personalized PageRank**：
- 從**特定節點**（seed）開始
- 計算每個節點相對於 seed 的「相關性分數」

---

## 🔢 PPR 分數怎麼算？

### 核心公式

```
π = (1 - α) × P × π + α × v
```

其中：
- `π` = PageRank 分數向量（每個節點的分數）
- `P` = 轉移矩陣（圖的鄰接矩陣，正規化後）
- `v` = teleport 向量（seed 分布，例如 reranked 的高分歌曲）
- `α` = damping factor（通常 0.15，控制跳回 seed 的機率）

### 迭代過程

```
1. 初始化：π⁽⁰⁾ = v（從 seed 開始）

2. 迭代：
   π⁽ᵗ⁺¹⁾ = (1 - α) × P × π⁽ᵗ⁾ + α × v

3. 重複步驟 2，直到收斂（π 不再變化）

4. 最終的 π 就是 PPR 分數
```

## 🎵 在推薦系統中的應用

### Step 1: 準備 Seed（種子節點）

從 Reranking 的高分歌曲中選擇 seed：

```
Reranking 後的前 10 名：
  - 歌曲 A: 分數 = 0.95
  - 歌曲 B: 分數 = 0.92
  - 歌曲 C: 分數 = 0.88
  ...

Teleport 向量 v：
  v[A] = 0.95 / (0.95 + 0.92 + 0.88 + ...) = 0.32
  v[B] = 0.92 / (總和) = 0.31
  v[C] = 0.88 / (總和) = 0.30
  ...
  v[其他] = 0.0
```

### Step 2: 在圖上做隨機游走

```
從 seed 節點開始：
  - 有 α 機率（例如 15%）跳回 seed
  - 有 (1-α) 機率（例如 85%）沿著圖的邊走

例如：
  從歌曲 A 開始
    ↓
  85% 機率：走到 A 的鄰居（20 首相似歌曲）
  15% 機率：跳回 seed（A, B, C...）
```

### Step 3: 迭代直到收斂

```
迭代 0: π = [0.32, 0.31, 0.30, 0, 0, ...]  (只有 seed 有分數)
迭代 1: π = [0.28, 0.27, 0.26, 0.05, 0.04, ...]  (seed 的鄰居開始有分數)
迭代 2: π = [0.25, 0.24, 0.23, 0.08, 0.07, ...]  (分數擴散更多)
...
迭代 N: π = [0.15, 0.12, 0.10, 0.08, 0.06, ...]  (收斂)
```

### Step 4: 最終分數

```
歌曲 A: PPR = 0.15（直接相關，seed）
歌曲 A1: PPR = 0.08（A 的鄰居，間接相關）
歌曲 B: PPR = 0.12（直接相關，seed）
歌曲 B1: PPR = 0.06（B 的鄰居，間接相關）
...
```

## 📐 具體例子

### 假設簡單的圖：

```
歌曲 A (seed) ←→ 歌曲 A1
    ↓
歌曲 A2 ←→ 歌曲 A3

歌曲 B (seed) ←→ 歌曲 B1
```

### 計算過程：

#### 初始化（t=0）：
```
v = [A: 0.5, B: 0.5, 其他: 0.0]
π⁽⁰⁾ = [0.5, 0.5, 0.0, 0.0, 0.0, 0.0]
```

#### 迭代 1（t=1）：
```
從 A 開始：
  - 85% 機率走到 A1, A2
  - 15% 機率跳回 A

從 B 開始：
  - 85% 機率走到 B1
  - 15% 機率跳回 B

π⁽¹⁾ = [0.15×0.5 + 0.85×0.25, 0.15×0.5 + 0.85×0.25, 0.85×0.25, ...]
      = [0.34, 0.34, 0.21, ...]
```

#### 迭代 2（t=2）：
```
分數繼續擴散...
π⁽²⁾ = [0.28, 0.28, 0.18, 0.12, ...]
```

#### 收斂後：
```
π = [0.15, 0.12, 0.08, 0.06, ...]
```

## 🔧 實作細節

### 1. 轉移矩陣 P

從圖的鄰接矩陣正規化得到：

```python
# 圖的鄰接矩陣（稀疏）
graph = load_npz('outputs/graph/song_graph.npz')  # (30246, 30246)

# 正規化：每行的和 = 1
row_sums = graph.sum(axis=1).A1
P = graph / row_sums[:, None]  # 正規化後的轉移矩陣
```

### 2. Teleport 向量 v

從 Reranking 的高分歌曲建立：

```python
# Reranking 的前 10 名
top_songs = ['song_A', 'song_B', 'song_C', ...]
top_scores = [0.95, 0.92, 0.88, ...]

# 正規化成分佈
v = np.zeros(num_songs)
for song_id, score in zip(top_songs, top_scores):
    idx = song_ids.index(song_id)
    v[idx] = score

v = v / v.sum()  # 正規化
```

### 3. 迭代計算

```python
alpha = 0.15  # damping factor
pi = v.copy()  # 初始化

for iteration in range(max_iter):
    pi_new = (1 - alpha) * P.dot(pi) + alpha * v
    
    # 檢查收斂
    if np.linalg.norm(pi_new - pi) < tolerance:
        break
    
    pi = pi_new
```

## 🎯 為什麼 PPR 分數是這樣？

### 分數的意義：

1. **直接相關（高分）**：
   - Seed 節點本身：分數最高（例如 0.15）
   - 因為有 15% 機率直接跳回 seed

2. **間接相關（中分）**：
   - Seed 的鄰居：分數中等（例如 0.08）
   - 因為可以從 seed 走一步到達

3. **更間接相關（低分）**：
   - Seed 的鄰居的鄰居：分數較低（例如 0.03）
   - 因為需要走兩步才能到達

### 分數大小取決於：

- **與 seed 的距離**：越近分數越高
- **圖的連接強度**：相似度越高，分數傳遞越多
- **Seed 的權重**：seed 本身的分數越高，擴散的分數越多

---

## 📊 完整流程總結

```
1. Reranking 得到高分歌曲（A, B, C...）
    ↓
2. 建立 teleport 向量 v（以 A, B, C 為 seed）
    ↓
3. 在圖上迭代計算 PPR：
   π = (1-α) × P × π + α × v
    ↓
4. 收斂後得到每個節點的 PPR 分數
    ↓
5. 排序，取 top-N 推薦
```

---

## 💡 關鍵參數

- **α (damping factor)**：通常 0.15
  - 越小：更傾向於在圖上走（探索更多）
  - 越大：更傾向於跳回 seed（更保守）

- **max_iter**：最大迭代次數，通常 100-1000

- **tolerance**：收斂閾值，通常 1e-6
