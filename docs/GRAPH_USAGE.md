# 歌曲相似度圖的用途（Graph Usage）

## 🎯 核心用途：Personalized PageRank (PPR) 推薦

這個圖是整個推薦系統的**核心組件**，用於 Personalized PageRank 演算法。

---

## 📊 圖的結構

```
30,246 首歌曲 = 30,246 個節點
604,920 條邊 = 歌曲之間的相似度連接
```

每首歌有 **20 個最相似的鄰居**，這些鄰居是根據 Emotion + Topic 向量計算出來的。

---

## 🚀 主要用途

### 1. **Personalized PageRank (PPR) 推薦** ⭐ 最重要

**流程**：
```
貼文 → BM25 找出 top-K 候選 → Reranking 得到高分歌曲
    ↓
以高分歌曲作為 "seed"（起點）
    ↓
在圖上做 Personalized PageRank
    ↓
分數會「擴散」到相似歌曲
    ↓
得到最終推薦列表
```

**為什麼需要圖？**
- 如果只靠 BM25 + Reranking，只能找到「直接相關」的歌曲
- 有了圖，可以找到「相關歌曲的相關歌曲」（二階、三階相似）
- 例如：歌曲 A 和 B 相似，B 和 C 相似 → 即使 A 和 C 沒有直接相似，也能被推薦

**例子**：
```
貼文："I feel sad and lonely"

Step 1: BM25 找到候選
  - 歌曲 A: "Fix You" (BM25=0.9, emotion=0.95)
  - 歌曲 B: "Someone Like You" (BM25=0.85, emotion=0.92)

Step 2: 在圖上做 PPR（以 A, B 為 seed）
  - 歌曲 A 的鄰居：A1, A2, A3... (20 首相似歌曲)
  - 歌曲 B 的鄰居：B1, B2, B3... (20 首相似歌曲)
  - PPR 會計算每首歌的「重要性分數」
  
Step 3: 最終推薦
  - 歌曲 A: PPR 分數 = 0.15
  - 歌曲 A1: PPR 分數 = 0.08 (A 的鄰居)
  - 歌曲 B: PPR 分數 = 0.12
  - 歌曲 B1: PPR 分數 = 0.06 (B 的鄰居)
  - ...
```

---

### 2. **歌曲發現（Music Discovery）**

**用途**：找到你可能會喜歡但還沒聽過的歌曲

**方法**：
- 從你喜歡的歌曲開始
- 在圖上「走訪」它的鄰居
- 發現新的相似歌曲

**例子**：
```
你喜歡：Taylor Swift 的 "Anti-Hero"
    ↓
圖會告訴你：這首歌的 20 個鄰居（相似歌曲）
    ↓
你可能會發現：
  - Olivia Rodrigo 的歌曲（相似情緒/主題）
  - Billie Eilish 的歌曲（相似風格）
  - 其他你還沒聽過的歌曲
```

---

### 3. **推薦多樣性（Diversity）**

**問題**：如果只用 BM25，可能推薦的都是「同一類型」的歌曲

**解決**：圖可以幫你找到「不同但相關」的歌曲

**例子**：
```
貼文："I'm tired"

BM25 可能推薦：
  - 10 首都是 "tired" 相關的歌曲（太單調）

PPR 在圖上擴散後：
  - 5 首 "tired" 相關
  - 3 首 "exhausted" 相關（相似但不同詞）
  - 2 首 "burnout" 相關（主題相似）
  → 推薦更多樣化！
```

---

### 4. **冷啟動問題（Cold Start）**

**問題**：新歌曲沒有歷史數據，很難推薦

**解決**：圖可以根據「相似度」推薦新歌曲

**例子**：
```
新歌曲 X 剛上架，沒有播放數據
    ↓
但圖告訴我們：X 和熱門歌曲 Y 很相似
    ↓
如果用戶喜歡 Y，就可以推薦 X
```

## 💡 為什麼圖比單純的相似度計算更好？

### 方法 1：不用圖（只算相似度）
```
貼文 → 找出最相似的 10 首歌 → 推薦
```
**問題**：
- 只能找到「直接相似」的歌曲
- 推薦可能很單調
- 無法發現「間接相關」的歌曲

### 方法 2：用圖（PPR）
```
貼文 → 找出候選 → 在圖上擴散 → 推薦
```
**優點**：
- ✅ 可以找到「相關歌曲的相關歌曲」
- ✅ 推薦更多樣化
- ✅ 能發現新歌曲
- ✅ 考慮全局結構（不只是兩兩相似）

## 📈 實際應用場景

### 場景 1：用戶輸入貼文
```
貼文："I'm feeling nostalgic"
    ↓
BM25 找到候選
    ↓
Reranking 排序
    ↓
PPR 在圖上擴散（使用這個圖）
    ↓
推薦：懷舊風格的歌曲 + 相似風格的歌曲
```

### 場景 2：播放列表推薦
```
用戶的播放列表：[歌曲 A, B, C]
    ↓
以 A, B, C 為 seed
    ↓
在圖上做 PPR
    ↓
推薦：與播放列表整體風格相似的歌曲
```

## 🎯 總結

這個圖的主要用途是：

1. **Personalized PageRank** - 在圖上做隨機游走，找到相關歌曲
2. **推薦多樣性** - 不只推薦直接相似的，也推薦間接相關的
3. **歌曲發現** - 幫助用戶發現新歌曲
4. **冷啟動** - 推薦新上架的歌曲

**沒有這個圖**：只能做簡單的相似度匹配  
**有了這個圖**：可以做複雜的圖譜推薦，效果更好！
